# Docs Guide

This directory is intentionally small.

## Active documents

- `development.md`: target-state architecture, interface constraints, and engineering direction.
- `progress.md`: current implementation truth, known gaps, and validated or blocked commands.
- `regression_matrix.md`: compact validation matrix for the combinations that matter today.

## Historical documents

- `archive/`: dated audit, remediation, and prep notes kept for traceability only.
- Files in `archive/` are not the source of truth for current scope or implementation status.

## Rules

- Update `progress.md` whenever capability status or validation confidence changes.
- Update `development.md` when target architecture, public module names, or interface contracts change.
- Use `bd` for work tracking. Do not turn `docs/` into a task tracker.
